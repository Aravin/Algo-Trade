# Algo Trade Client — AI Context (llm.txt)

> Domain: Automated & Semi-Automated Algorithmic Trading Client (V5 Strategy System)
> Stack: React 19, TypeScript, Vite 8, Cloudflare Workers, Cloudflare D1 (SQLite), Recharts, Radix UI, Tailwind CSS v4.

---

## 1. Project Overview & Architecture
`app/client` is the client-side trading dashboard and serverless edge proxy layer for the Algo-Trade system.
It features browser-executed trading logic (V5 Strategy Engine) synchronized with a Cloudflare Worker edge backend.

- **Vite React UI (`src/`)**: Renders real-time strategy monitors, score breakdowns, threshold optimizers, paper trading management, broker account links, and interactive charts.
- **Cloudflare Worker (`worker/`)**: Edge API proxy for Upstox API v2/v3, Paper Trading persistent ledger (D1 database `algo-trade-paper`), Auth0 token validation, and Client State persistence (`client_state` table).
- **Execution Engine (`src/lib/strategyEngine.ts`)**: Evaluates real-time market signals (VWAP, Supertrend, EMA 9/21, Bollinger Bands, RSI 14, PCR, FII/DII, Market Breadth, VIX, Global Markets) to produce directional buy/sell signals, score breakdowns, and execution triggers.

## 3. Data Flow & Communication Lifecycle

```
[Upstox REST / Market Proxy] ──> [Worker /api/market/*] ──> [marketService.ts]
                                                                    │
                                                                    ▼
[React UI: useStrategyBot] <── [strategyEngine.ts] <── [Tick Data & Indicators]
            │
            ├─► [Paper Trading: /api/paper/*] ──► [D1 Database: algo-trade-paper]
            │
            └─► [Live Orders: /api/order/place] ──► [Upstox Live API]
```

## 4. Key API Endpoints (Cloudflare Worker)

| Endpoint | Method | Auth Required | Description |
| :--- | :---: | :---: | :--- |
| `/api/broker/upstox/token` | POST | No | Exchanges OAuth code for Upstox access token |
| `/api/market/quotes` | POST | No | Fetches live market quotes for instruments |
| `/api/market/option-chain` | POST | No | Fetches complete option chain with greeks & PCR |
| `/api/market/vix` | POST | No | Fetches India VIX volatility metric |
| `/api/market/breadth` | POST | No | Fetches market advance/decline breadth |
| `/api/market/upstox/news` | POST | No | Fetches live market news alerts |
| `/api/market/global-sentiment` | GET/POST | No | Fetches global market indices sentiment |
| `/api/client-state` | GET/PUT | Yes (Auth0) | Persists strategy config & account settings to D1 |
| `/api/paper/account` | GET | Yes (Auth0) | Retrieves paper trading balance & summary |
| `/api/paper/trades/enter` | POST | Yes (Auth0) | Executes a paper trade entry into D1 |
| `/api/paper/trades/exit` | POST | Yes (Auth0) | Executes a paper trade exit & records PnL |
| `/api/order/place` | POST | Yes (Auth0) | Places live order to Upstox broker |

## 5. Key Commands

- `yarn dev` — Launch Vite dev server locally.
- `yarn build` — Typecheck (`tsc -b`) and build Vite production bundle.
- `yarn test` — Run Vitest unit tests (`src/lib/__tests__`).
- `yarn validate` — Run typecheck, lint, format check, and unit tests.
- `yarn deploy` — Build client & deploy Cloudflare Worker.
- `yarn wrangler d1 migrations apply algo-trade-paper --local` — Apply local D1 schema.
